Map objects in BayesX

In BayesX map objects are used to handle, store and visualize geographical information. In the current version map objects serve as auxiliary objects for regression analyses, where the effect of spatial covariates on a dependent variable can be modelled via Markov random field priors, but they also allow the visualization of any geographical information associated with the map that is provided in a dataset object. Map objects can be created from boundary files or graph files, compare section 5 of the manual for a description of these file formats.

Converting shape files to boundary files

To convert a shape file into a boundary file, you may use the R-function shp2bnd. Shp2bnd uses facilities provided in the packages shapefiles and maptools to extract the required information from a shape file and writes the boundary information into a specified file (Thanks to Frank Sagerer for the reference to the shapefiles-package!).

An exemplary R-session might look like this:

R> source("c:\\bayesx\\sfunctions\\shp2bnd.r")
R> shp2bnd(shpname="c:\\temp\\gm", regionnames=1:16, bndname="c:\\temp\\test.bnd", replace=T)

To check whether the specified boundary file was created correctly, type

R> source("c:\\bayesx\\sfunctions\\readbndfile.r")
R> source("c:\\bayesx\\sfunctions\\drawmap.r")

R> readbndfile("c:\\temp\\test.bnd", "m")
R> col <- runif(16)
R> nam <- levels(as.factor(names(m)))
R> drawmap(map=m, plotvar=col, regionvar=nam)

To reproduce the example, download the shapefile gm.zip and extract it to a directory (c:\temp in the example above). Download the function shp2bnd and store it in the directory 'sfunctions' (a subdirectory of the BayesX installation directory, c:\bayesx in the example above). Now you can paste the example code to the R prompt (after changing the paths according to your specifications).

Available boundary and graph files

On this page we provide boundary files as well as graph files for some maps that have already been used with BayesX. We also request users of BayesX to provide us their maps so that we can make them available on this page.

Germany:

For the map of Germany we provide several maps, since between 1993 and 1996 the boundaries and number of districts changed in the eastern part of Germany. In 2001 the two districts of Hannover were united. Moreover the island Rügen causes the map to be disconnected resulting in problems when estimating structured spatial effects and therefore we provide one version including Rügen and one version without Rügen. The regioncodes are given by the key numbers of the administrative districts.

  • Western part of Germany
    • Boundary file
    • Graph file
  • Germany before 1993 including Rügen (disconnected map)
    • Boundary file
    • Graph file
  • Germany before 1993 without Rügen
    • Boundary file
    • Graph file
  • Germany between 1996 and 2001 including Rügen (disconnected map)
    • Boundary file
    • Graph file
  • Germany between 1996 and 2001 without Rügen
    • Boundary file
    • Graph file
  • Germany since 2001 including Rügen (disconnected map)
    • Boundary file
    • Graph file
  • Germany since 2001 without Rügen
    • Boundary file
    • Graph file
